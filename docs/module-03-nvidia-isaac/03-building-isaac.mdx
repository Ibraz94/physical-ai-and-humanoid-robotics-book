### **Importing URDF into Isaac Sim**

  

Let's import the mobile robot from Module 2.

  

**Using URDF Importer:**

  

1. Isaac Utils → Workflows → URDF Importer

2. Browse to your `mobile_robot.urdf`

3. Configure import options:

-  **Import as Instance:** No

-  **Fix Base Link:** No (robot should move)

-  **Joint Drive Type:** Velocity

-  **Self Collision:** Enable

4. Click **Import**

  

The robot appears with proper materials, joints, and physics!

  

### **Configuring ROS 2 Bridge**

  

Isaac Sim has built-in ROS 2 support—no external bridge needed.

  

**Enable ROS 2:**

  

1. Window → Extensions

2. Search "ROS2"

3. Enable "omni.isaac.ros2_bridge"

  

**Configure bridge:**

  

1. Isaac Utils → Workflows → ROS2

2. Set Domain ID: 0

3. Enable "Auto-start bridge"

  

### **Adding Differential Drive Controller**

  

1. Select robot in Stage

2. Right-click → Create → Isaac → Controllers → Differential Controller

3. Configure:

-  **Left Wheel Joint:** left_wheel_joint

-  **Right Wheel Joint:** right_wheel_joint

-  **Wheel Radius:** 0.1

-  **Wheel Distance:** 0.5

  

### **Adding Sensors**

  

**Camera:**

  

1. Create → Camera

2. Position on robot front

3. Isaac Utils → ROS Components → RGB Camera

4. Configure:

- Topic: `/camera/image_raw`

- Frame rate: 30

- Resolution: 640x480

  

**LiDAR:**

  

1. Create → Isaac → Sensors → LiDAR → Rotating Lidar

2. Position on robot top

3. Configure:

- Horizontal FOV: 360°

- Range: 10m

- ROS Topic: `/scan`

  

**IMU:**

  

1. Isaac Utils → Sensors → IMU Sensor

2. Attach to robot base

3. Topic: `/imu`

  

### **Testing the Robot**

  

Press **Play** in Isaac Sim.

  

In a terminal:

  

```bash

# Check topics

ros2  topic  list

  

# Should see /cmd_vel, /odom, /camera/image_raw, /scan, /imu

  

# Control robot

ros2  topic  pub  /cmd_vel  geometry_msgs/msg/Twist  "{linear: {x: 0.5}, angular: {z: 0.0}}"

```

  

The robot moves in Isaac Sim with photorealistic rendering!

  

**View camera feed:**

  

```bash

ros2  run  rqt_image_view  rqt_image_view

```

  

Select `/camera/image_raw` to see the robot's perspective.

  

### **Creating a Custom Environment**

  

Let's build a simple navigation environment.

  

**Add floor:**

  

1. Create → Shape → Cube

2. Scale: (20, 20, 0.1)

3. Position: (0, 0, -0.05)

4. Make static: Property panel → Physics → Rigid Body → Static

  

**Add walls:**

  

```python

# In Isaac Sim Python console (Window → Script Editor)

from pxr import UsdGeom, Gf

  

stage = omni.usd.get_context().get_stage()

  

# Create wall function

def  create_wall(path, position, size):

cube = UsdGeom.Cube.Define(stage, path)

cube.GetSizeAttr().Set(1.0)

cube.AddTranslateOp().Set(Gf.Vec3d(position[0], position[1], position[2]))

cube.AddScaleOp().Set(Gf.Vec3d(size[0], size[1], size[2]))

return cube

  

# Create walls

create_wall("/World/wall_north", (0, 10, 1.5), (20, 0.2, 3))

create_wall("/World/wall_south", (0, -10, 1.5), (20, 0.2, 3))

create_wall("/World/wall_east", (10, 0, 1.5), (0.2, 20, 3))

create_wall("/World/wall_west", (-10, 0, 1.5), (0.2, 20, 3))

```

  

**Add obstacles:**

  

```python

import random

  

for i in  range(10):

pos = (random.uniform(-8, 8), random.uniform(-8, 8), 0.5)

size = (random.uniform(0.5, 1.5), random.uniform(0.5, 1.5), 1.0)

create_wall(f"/World/obstacle_{i}", pos, size)

```

  

Now you have a custom environment for testing!
